# PowerTech — рабочая заметка (handoff)

## Где что (ВАЖНО)
- **Ветка `dev` = настоящий сайт. Работаем здесь.** (эта ветка)
- **Ветка `main` = заглушка «Скоро открытие».** Это то, что сейчас видят посетители на powertech.am.
- Сайт **намеренно скрыт** от публики, пока не готов (люди уже заходили на домен).

## Как устроен деплой
- Хостинг: **GitHub Pages**, домен **powertech.am** (файл `CNAME`).
- Pages отдаёт на домен **только ветку `main`**. Ветка `dev` **нигде не публикуется** → в ней можно спокойно править, посетители этого не видят.
- Никаких GitHub Actions — публикация идёт прямо из ветки.

## Как править
1. Работать в ветке **`dev`** (в Claude Code на web выбрать эту ветку).
2. Коммитить в `dev` — на живой сайт это НЕ влияет.
3. Предпросмотр: локальный сервер (`npx serve .`) или подключить preview-хостинг (Netlify / Cloudflare Pages) к ветке `dev` — он даст отдельную ссылку, не трогая powertech.am.

## Как ОПУБЛИКОВАТЬ, когда всё готово
Перенести содержимое `dev` на `main` (заглушка заменится настоящим сайтом):
```
git checkout main
git checkout dev -- .
git add -A
git commit -m "Publish full site"
git push origin main
```
Через ~1–2 минуты powertech.am покажет полный сайт. (Альтернатива: `git checkout main && git reset --hard dev && git push --force origin main`.)

## Состояние сайта на момент паузы
- Языки: `index.html` = HY (армянский), `ru/`, `en/`. Общий `style.css`, скрипты в `assets/`.
- Версии ассетов (кэш-бастинг `?v=`): **style.css v=251**, site-fx v=22, hero-scope v=7, site-award v=12.
- Порядок секций: Hero → **Экономика** (калькулятор стоимости, 4 сценария-таба) → Сферы → Мониторинг → Параметры → Отчёт → Подход → О компании → Контакт.
- Карточная обработка (беж `#DCD8D3` + белая рамка 14px, без тени) применена к форме заявки, секции «Экономика» и «Отчёт».
- Форма заявки — Web3Forms (доставка подтверждена).

## Не публиковать в прод случайно
В `.gitignore` стоит игнор черновиков. Не коммитить `variants-*.html`, `portfolio/`, `three.min.js`, `ru/zachem/` — это эксперименты, не часть сайта.
